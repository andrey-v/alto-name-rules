Именование таблиц и полей

    Таблица
        Имя таблицы базы данных описывается фразой, поясняющей смысл хранимых значений в единственном числе. То есть
        если таблица хранит данные о пользователях, то таблица должна называться user, а не users. Имя таблицы может
        иметь префикс, указанный в настройках CMS. Имя таблицы записывается латинскими буквами нижнего регистра, а
        знак нижнего подчеркивания разделяет слова имени и отделяет префикс.

        Если таблица создана и испоьзуется плагином, то имя таблицы должно начинаться с имени плагина;

        Пример:
            prefix_user - таблица, хранящая записи о пользователях;
            prefix_pluginname_manager - таблица, хранящая записи о менеджерах плагина с именем pluginname;

    Поле
        Имя поля таблицы должно содержать префикс имени таблицы, указанного без префикса базы данных и префикса
        плагина. Если поле используется для связи с другой таблицей, то его имя должно содержать, в качестве
        дополнительного префикса (также без префикса БД), имя связанной таблицы.

        Пример:
            manager_id (таблица_поле)                       - идентификатор менеджера (ключ);
            manager_user_id (таблица_связаннаятаблица_поле) - идентификтор пользователя;
            
    Строка sql-запроса
        Один метод маппера содержит один sql-запрос.
        Переменная для хранения строки sql-запроса называется $sql.
        Строка sql-запроса обрамляется двойными кавычками.
        
    Псевдонимы таблиц
        Псевдонимы таблиц именуются одной буквой, которая является первой в имени сущностией таблицы . Например, если 
        выборка идет из таблицы "prefix_blog", то псевдоним будет "b" - SELECT b.* FROM prefix_blog AS b; 
        Если же в запросе есть совпадение первых букв имен сущностей таблицы, то псевдоним строится от имени сущности 
        выбросом глассных букв, например, если идет выборка по таблицам "geo_country" и "geo_city", то псевдонимы будут 
        следующие - "cntr" и "сt"
